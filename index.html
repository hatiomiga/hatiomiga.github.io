mysql命令行基本操作

基本操作

  • 创建数据库

    CREATE DATABASE IF NOT EXISTS `数据库名`

  • 删除数据库

    DROP DATABASE IF EXISTS `数据库名`

  • 使用数据库

    USE `数据库名`

  • 查看所有数据库

    SHOW DATABASES

  • 创建数据库表

  • 查看已存在数据库的语句

    SHOW CREATE DATABASE `表名`

-查看数据表的定义语句

SHOW CREATE TABLE `表名`

  • 查看表的具体结构

    DESC `表名`

  • 修改表名

    ALTER TABLE `旧表名` RENAME `新表名`

  • 增加表的字段

    ALTER TABLE `表名` ADD `字段名` 字段属性

  • 修改表的字段(1.修改属性，2.重命名)

    1.ALTER TABLE `表名` MODIFY `字段名` 字段属性
    2.ALTER TABLE `表名` CHANGE `旧字段名` `新字段名` 字段属性

  • 删除表的字段

    ALTER TABLE `表名` DROP `字段名`

  • 删除表

    DROP TABLE IF EXISTS `表名`

#外键(了解即可)

  • 外键

  • 创建表时没有外键关系

Java IO

java IO流详解

基本概念

1.流的输入和输出是相对内存的。
从内存出来为输出，
到内存去为输入。
(比如我从你的电脑拷贝文件这一行为，对于我的内存来说是输入
而对于你的内存则为输出）

2.流根据读取数据的方式分为字节流和字符流，字节流按照字节读取，
字符流按照字符读取，一次读取2个字节

3.字节流适合读取视频，声音，图片等二进制文件。字符流适合读取纯文本文件。

4.java语言中所有字节流都是以Stream结尾，
而所有的字符流都含有Reader或Writer。

各种流的概括

  • 以File开头的与文件相关
  • 以Buffered开头的自带缓冲区
  • 以Object开头的专门读取Java对象
  • InputStreamReader和OutputStreamWriter为转换流(字节流转换成字符流)
